## Findings

### QA-1 · New Loop form: Provider select desyncs from state — **Bug / Medium** → issue #141

- **Scenario:** 8 (Loops) / 9 (provider reconciliation)
- **Steps:** Open `/project/:id/loops/new` with a `settings.provider` that has no key
  (here `anthropic`; authenticated = `openai, openrouter, nvidia, ollama`).
- **Expected:** Provider select shows the provider actually in state (anthropic), or resets to a valid authenticated one.
- **Actual:** Select displays **OpenAI** (first authenticated option) while state stays `anthropic`;
  Model select shows `claude-sonnet-4-6` (Anthropic) → visible provider/model mismatch, and submitting
  pins a provider the UI never showed.
- **Root cause:** `src/renderer/src/views/NewLoopForm.tsx` ~L258–275 — fallback `<option value={provider}>`
  is rendered only when `authProviders.length === 0`. The Model select handles the same case correctly (L69
  prepends the current value); the Provider select should mirror that.
- **Evidence:** `qa-reports/assets/r-loops-new.png`; live `providers.list()` auth dump in issue #141.

### QA-2 · Composer Start button stays enabled on empty/whitespace prompt — **UX / Low** → issue #142

- **Scenario:** 4 (Compose / start agent)
- **Steps:** Compose view, empty or whitespace-only prompt, no attachment → "Start agent" is enabled; clicking is a no-op.
- **Expected:** Button disabled when there's nothing to send.
- **Actual:** `disabled={starting}` only; `start()` early-returns (L73) so click does nothing, but the button looks active.
- **Suggested fix:** `disabled={starting || (!text.trim() && att.items.length === 0)}` (`Compose.tsx:197`).
- **Evidence:** `qa-reports/assets/r-compose.png`.
